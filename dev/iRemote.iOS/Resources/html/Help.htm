iRemote Suite is iPhone application that allows you to manipulate your workstation or notebook.

1. Requirements

Network neighborhood

Network neighborhood is the primary thing you need to take an advantage of using this software. At first, you need local area network to be able to connect your desktop or notebook. And finally, you need WiFi router to be able to connect your iPhone to your local network.

The scenario may be easier, of course, if you have a notebook with embedded WiFi module which is capable to create Ad Hoc network. In this case you can connect your iPhone to notebook directly without mediators.

Host application

Before you launch your mobile client you have to download and install host application, which is called Remote PC Suite on your desktop system. You may found latest binaries for your platform at
http://www.scientific-soft.com/mobile/apps/iremote/
Note: Binaries for your desktop system are always FREE. And what is much more important is that you should not download binaries from third-party mirrors. The only authorized distributor of Remote PC Suite is Scientific Software. Please, use authorized mirror to protect your system.

2. Connecting to Host

The only way to connect to your host is to take an advantage of TCP/IP networking. It means that you have to specify IP address and PORT of your desktop to establish the connection.

Remote PC Suite provides you a way to acquire your IP address easily. It is displayed at the bottom of Remote PC Suite main window.


Part of host application window with ip combo box.

Advanced
The software uses port number 8444 by default. It may occur so, that another software (on your desktop) uses the same port number. In this case you will be prompted with an error when you launch Remote PC Suite. You will have to choose another port value and specify it while logging in with iRemote Suite.


iRemote login page with ip and port populated.

Press ‘Catch’ button to establish connection with host application which should be launched on your workstation.

It is now possible to remember your device on the desktop. This feature allows you to connect to a host remotely. You will need to enter password on the device just once.


Password request in iRemote.

After successful connection attempt, you will have this host on the list and will be able to connect to it 'in a click' without any confirmations on desktop side.


List of known hosts.

Here you can select any particular host to connect to it immediately. You are also able to remove hosts from the list in standard iPhone way.

You can get one of the following erros, when attempting to extablish connection with the host:

  • Failed to connect to host.

    This error occurs, when target host is not accessible. Check whether IP address and port values are correct and ensure that Remote PC Suite is launched. You can also try to restart Remote PC Suite on the host.

  • Connection is terminated by host.

    Host application terminated connection for some reason. This error occurs, when user disconnects device by means of Remote PC Suite, or Remote PC Suite is terminated.

  • Limit of connections is reached.

    The only device is allowed to be connected to the host at the moment. You will get this error, if you try to connect a device, when another device is already connected.

  • Wrong device IP address.

    If you create a rule for your device (remember it on the host), the device is associated with its cureent IP address. If device IP address is changed for some reason, you have to proceed to Remote PC->User's Manager->Your Device and modify IP address. If device IP is frequently changed, you can turn this feature off at the same place. Just leave 'Associate with IP address' checkbox blank.

3. TouchPad

Pan Feature

Use pan gestures to move mouse cursor over the screen of your desktop. Mouse events are inverted. This means the following. Imagine, that when you move you finger along the screen of your device you literally move visible area along your desktop screen.

Scroll Feature

Touch screen with two fingers simultaneously and pan to scroll active window vertically. You are also able to scroll any window horizontally if you are running Windows Vista (or higher) on your desktop.

Zoom Feature

You are able to zoom in and zoom out using enther +/- controls (at the rop of the screen) or two fingers pinch guesture. This feature allows you to see the entire screen or certain part of it, as close as it is necessary.


Sceeen zooming in action.

Landscape Mode

This feature allows you to control desktop in landscape mode. Landscape mode is only enabled on Touch Pad and iKeyboard screens. Note that you cannot navigate between screens in landscape mode. You should turn iPhone/iPod into portrait mode to see tab bar at the bottom of the screen.


Viewing desktop in landscape mode.

Clicks Feature

Use buttons to simulate mouse clicks. It is now possible to tap touch pad to simulate mouse click. Mouse click will be sent to current cursor position.

Note: Mouse and keyboard buttons behave as natural ones. Application distinguishes button down and button up events. This allows you to drag windows and select regions combining button down and pan gestures.


Selecting area of the desktop.

Standard Keyboard

Since iRemote 1.2.2 there is no separate screen for standard keyboard. You can now toggle keyboard directly on Touch Pad screen, when necessary. Just use button at the top right corner of the screen to show/hide keyboard. You may use standart iPhone keyboard to input everything you want using any international keyboard layout.


Text input is fast with standard keyboard.

4. Shell

This feature provides you with a chance to get access to native desktop system command interpreter from your iPhone. You are able to configure your system, launch applications and do lots of other things by means of command interpreter. Native interpreter for Mac platform is Bash. Windows runs cmd.exe for the same purposes, but it's functionality is much more limited.

It is almost impossible to cover all shell scripting topics for both Mac and Windows platforms in a single manual. If you are totally unfamiliar with command prompts, you can type 'dir' to get the contents of current directory under Windows system, or 'ls' to do the same thing under Mac.


Pinging google.com.

You can also refer to the following resources:

  • http://en.wikipedia.org/wiki/Bash
  • http://en.wikipedia.org/wiki/ Command_Prompt_(Windows)

Important notes:

  • Windows. When attempting to launch an application by means of iRemote you must use 'start $app name$' syntax instead of just '$app name$'.

  • Some shell commands (such as 'ping') do not return execution to the commands interpreter immediately. If you run one of such commands and disconeect from the host, it (command) will continue execution in the background. When you extablish connection again, you will see new messages, coming from the shell. Note, that you will get only new messages, when connection is extablished. Remote PC Suite does not caches messages, when there are no connected clients.

    You can use 'Ctrl-C' button (at top left corner) to terminate such command. This button actually simulates Ctrl-C combination.

  • Each connected device gets access to desktop shell without a warning of any kind.

  • Network interaction between Remote PC Suite and iRemote Suite is not encrypted. This means that, all your commands and server responses are transferred as is.

  • A number of lines in shell view on the device is limited to 128. This limitations is caused by performance considerations. User interface of iRemote may become sluggish, if shell command outputs huge amount of data (try 'man bash' on MacOSX for an example). This is generally caused by shell view control, and will be fixed in future iRemote releases.

6. Full Size Keyboard

This feature provides you with a chance to do almost everything you can do with your regular keyboard. Of course, due to its size it may not efficiently replace desktop one.

7. Settings

Pan Mode

Default mode assumes that mouse cursor moves in the same direction finger moves along the screen surface.


Inverse panning (default mode).

Turning this mode off, allows you to literally grag screen content along the device screen.


Direct panning.

The best way to understand the difference and to pick the most appropriate mode is to try both!

Autorotation

This option allows you to disable touch pad autorotation, when device orientation changes.

8. Sleep Mode

iRemote Suite terminates connection with the host as soon as device goes to sleep mode. When device wakes up, the application attempts to reestablish the connection. You will not prompted for confirmation in this case for convenience. Application exchanges security cookie with the host any time, when you prove authenticaiton request clicking 'Allow' button in Remote PC, and uses this cookie to reconnect automatically.

iRemote Suite does not prevents for the device to go to sleep to increase battery life.

9. Limitations

  • iRemote Suite (by means of Host Application) does not simulate DirectInput events.
  • You may connect the only device running iRemote Suite to your host at the time.
